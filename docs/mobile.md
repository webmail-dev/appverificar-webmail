# Lectura movil

## Objetivo

La documentacion esta escrita para leerse comoda desde GitHub Desktop, GitHub Mobile y VitePress.

## Criterios aplicados

- Indices cortos al inicio de documentos largos.
- Tablas reducidas o reemplazadas por listas cuando podian crecer demasiado.
- Bloques de codigo cortos y copiables.
- Titulos directos.
- Secciones separadas por tema.
- Diagramas Mermaid simples.
- Enlaces relativos compatibles con GitHub.

## Reglas para mantenerla

- Evitar tablas con muchas columnas.
- Evitar lineas de comandos muy largas.
- Dividir guias extensas en documentos por tema.
- Mantener listas de pasos en bloques cortos.
- Usar rutas y nombres de archivo reales.
- No documentar servicios que no existan en el repositorio.

## Validacion manual recomendada

Antes de publicar cambios grandes:

1. Abrir `README.md` en GitHub Desktop.
2. Abrir `README.md` en GitHub Mobile.
3. Revisar `docs/deployment.md`.
4. Revisar `docs/dokploy.md`.
5. Confirmar que los bloques de codigo no obligan a desplazamiento horizontal excesivo.
6. Confirmar que los diagramas Mermaid no reemplazan instrucciones textuales.
